# Definition of Done (DoD) — Lab 7

## DoD — Requirement (REQ-###)
A requirement is considered "Done" when:
1. Possui um ID estável, título claro e tipo definido (Funcional ou Não-Funcional).
2. A descrição está completa e não descreve detalhes de implementação tecnológica (o "quê" e não o "como").
3. Os impactos de variantes (Security, Audit, etc.) estão explicitamente identificados.
4. Existem entre 2 a 4 Critérios de Aceitação (AC) testáveis.
5. Foi validado em sessão de Role Play com Stakeholders e DevTeam.
6. Eventuais conflitos com outros requisitos foram resolvidos e documentados.
7. O método de verificação (Demo, Teste, Review) está definido.
8. Está revisto gramaticalmente e formatado corretamente no repositório.

## DoD — User Story (US-###)
A user story is considered "Done" when:
1. Segue a estrutura "As a [role], I want [action], so that [value]".
2. É pequena o suficiente para ser implementada num único ciclo (Sprint/Iteração).
3. Todos os critérios de aceitação (AC) foram testados e passaram.
4. O código cumpre os padrões de qualidade e passou em Code Review (se aplicável).
5. A documentação técnica ou manual de utilizador foi atualizada.
6. Não existem defeitos críticos (bugs) abertos associados à história.
7. Foi demonstrada com sucesso ao Product Owner/Stakeholder.
