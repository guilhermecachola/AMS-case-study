# Glossary

**Intake** – The initial submission of project information for evaluation.

**Validation** – The process of checking whether submitted data meets defined rules.

**Mandatory Field** – A required field that must be completed before submission.

**Cross-field Validation** – A rule that checks consistency between two or more fields.

**Incomplete State** – Status assigned when mandatory data is missing.

**Invalid State** – Status assigned when validation rules fail.

**Ready to Decide** – Status assigned when all validation rules pass.

**Evidence** – Supporting documentation or justification provided with the intake.

**Requirement (REQ)** – A formal statement describing system behavior or constraint.

**Acceptance Criteria** – Conditions that must be satisfied for a requirement to be accepted.

**Use Case** – A structured description of system interaction.

**Test Case** – A defined procedure to verify system behavior.
